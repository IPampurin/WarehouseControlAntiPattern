## Файлы для WarehouseControl — CRUD склада с историей и ролями (антипаттерн триггеры СУБД)  

### 📋 Описание проекта  

Любой склад — это не просто список товаров. Это движущаяся система, где кто-то добавляет,  
кто-то списывает, кто-то правит остатки, и важно понимать кто, что, когда и зачем сделал.  
Кроме того, не все сотрудники должны иметь одинаковые права: кладовщик может редактировать,  
менеджер — только смотреть, а аудитор — видеть историю.  

В этом проекте приведен пример мини-системы для учёта товаров на складе: с базовыми  
CRUD-операциями, логированием всех изменений и ролевой моделью доступа. Это типичный  
пример корпоративного backend-сервиса, каких много в логистике, производстве и торговле.  

Но реализация проекта намерено выполнена на антипаттерне - триггерах внутри СУБД.  
Проект является примером того, что это такое и отвечает на вопрос почему никогда  
не надо так делать.  

### 🖥️ Возможности  

HTTP-методы (API):  

- POST /login — получение JWT-токена по роли (admin/manager/viewer)  
- POST /items — создание нового товара (требуются права admin/manager)  
- GET /items — получение списка всех товаров (доступно всем)  
- PUT /items/{id} — обновление товара (admin/manager)  
- DELETE /items/{id} — удаление товара (только admin)  
- GET /items/{id}/history — история изменений конкретного товара (только admin)  
- GET /history — глобальная история всех товаров с фильтрацией по дате, пользователю, действию (admin)  
- GET /history/compare?from={id}&to={id} — сравнение двух версий товара (admin)  
- GET /export/items/csv — экспорт всех товаров в CSV (доступно всем)  
- GET /export/history/csv — экспорт отфильтрованной истории в CSV (admin)  

Ролевая модель:  

- admin — полный доступ: создание, редактирование, удаление, просмотр истории, экспорт  
- manager — может создавать и редактировать товары, но не удалять и не видеть историю  
- viewer — только просмотр списка товаров  

Веб-интерфейс:  

- Главная страница с выбором роли (для демонстрации).
- Страницы для каждой роли с соответствующими правами:
    - для admin — вкладки «Товары» и «История», возможность добавлять, редактировать, удалять товары, просматривать историю, сравнивать версии, экспортировать CSV.
    - для manager — только управление товарами (без истории и удаления).
    - для viewer — только просмотр списка товаров.
- Фильтрация истории по дате, пользователю, действию.
- Сравнение двух записей истории с отображением изменений полей.
- Экспорт данных в CSV.

### 🚀 Быстрый старт  

**Требования:**  

- Docker  
- Свободные порты:  

    8081 — веб-интерфейс и API  
    5432 — PostgreSQL  

**Docker-образы**  

- postgres:16  

**Запуск системы:**  

    cd ./ && docker compose up  


- После успешного запуска веб-интерфейс будет доступен по адресу: http://localhost:8081  

### ⚙️ Конфигурация  

Все настройки задаются через файл .env в корне проекта.

    ## переменные сервера и api
    SERVICE_HOST_NAME=0.0.0.0          # хост, на котором слушает сервис
    SERVICE_PORT=8081                  # порт сервера
    GIN_MODE=debug                     # режим логов Gin (debug / release)

    ## переменные базы данных
    DB_HOST_NAME=postgres              # имя контейнера PostgreSQL в сети Docker
    DB_PORT=5432                       # порт PostgreSQL
    DB_NAME=postgres                   # имя базы данных
    DB_PASSWORD=postgres               # пароль
    DB_USER=postgres                   # пользователь

    ## пример ключа подписи JWT
    SECRET_KEY_JWT=secret-key-jwt      # очень секретный ключ для подписи токенов
